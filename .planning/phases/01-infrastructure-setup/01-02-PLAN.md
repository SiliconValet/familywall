<task type="auto">
  <name>Task 2: Create graceful restart script with browser and backend coordination</name>
  <files>scripts/restart-kiosk.sh</files>
  <action>
    Create scripts/restart-kiosk.sh following research Example 3 (Complete Restart Script with Error Handling):
    - Add shebang: #!/bin/bash
    - Set error handling: set -euo pipefail
    - Define LOG_FILE=/home/pi/familywall/logs/restart.log
    - Create log() function that timestamps and writes to log file
    - Export DISPLAY=:0 and XAUTHORITY=/home/pi/.Xauthority (for user pi session)
    - Log "Starting kiosk restart"
    - Gracefully stop Chromium: pkill -SIGTERM chromium-browser (ignore errors)
    - Wait up to 10 seconds for clean exit (loop checking pgrep chromium-browser)
    - If still running after 10s, force kill: pkill -SIGKILL chromium-browser
    - Log reload backend: pm2 reload ecosystem.config.js --update-env
    - Sleep 3 seconds for backend to be ready
    - Verify backend is online: pm2 status | grep -q "online" or exit 1
    - Restart Chromium with same flags as autostart (research Example 3 has exact command)
    - Log "Kiosk restart complete"
    - Make script executable: chmod +x

    IMPORTANT: Use SIGTERM first (graceful), wait, then SIGKILL as fallback. Per research Pitfall 5: "Always send SIGTERM first, wait up to 10 seconds, only then use SIGKILL". This prevents Chromium profile corruption. Also use `pm2 reload` (graceful) not `pm2 restart` (immediate) — prevents SQLite locking issues per research Pitfall 8.
  </action>
  <verify>
    <automated>test -x scripts/restart-kiosk.sh && grep -q "pkill -SIGTERM chromium-browser" scripts/restart-kiosk.sh && grep -q "pm2 reload" scripts/restart-kiosk.sh</automated>
  </verify>
  <done>
    restart-kiosk.sh is executable, contains graceful SIGTERM→wait→SIGKILL sequence for Chromium, uses pm2 reload for backend, logs all actions with timestamps, verifies backend is online before restarting browser.
  </done>
