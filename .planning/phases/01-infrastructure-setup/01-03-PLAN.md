<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify infrastructure stack on Raspberry Pi hardware</name>
  <what-built>
    Complete infrastructure stack deployed to Raspberry Pi with:
    - npm workspaces monorepo structure
    - PM2-managed Fastify backend with 512MB memory limit
    - React client building to server/public/
    - labwc kiosk configuration for Chromium autostart
    - labwc idle configuration preventing screen blanking
    - systemd timer scheduling daily 3am restart
    - Graceful restart script coordinating browser and backend
  </what-built>
  <how-to-verify>
    **Boot Verification (INFRA-01):**
    1. Reboot the Raspberry Pi: `sudo reboot`
    2. Wait for boot to complete (~30-60 seconds)
    3. Observe: Chromium should open full-screen in kiosk mode automatically
    4. Verify: No browser UI visible (no address bar, tabs, or window controls)
    5. Verify: Page displays "FamilyWall - Loading..." or similar React content
    6. Verify: URL bar accessible via F11 toggle should show http://localhost:3000

    **Screen Blanking Prevention (INFRA-03):**
    7. Wait 5-10 minutes without touching the screen
    8. Observe: Screen should remain on and NOT blank or enter power save mode
    9. Verify: Touch the screen — should respond immediately (not waking from sleep)

    **PM2 Backend Management (INFRA-04):**
    10. SSH into Pi: `ssh pi@raspberrypi.local`
    11. Check PM2 status: `pm2 list`
    12. Verify: familywall-backend shows status "online"
    13. Check memory limit: `pm2 show familywall-backend | grep "max memory"`
    14. Verify: Shows "512M" memory limit
    15. Test crash recovery: `pm2 stop familywall-backend && sleep 2 && pm2 list`
    16. Verify: Backend auto-restarts and returns to "online" status
    17. Test memory limit (optional): `pm2 trigger familywall-backend memory-leak` if implemented
    18. Check logs directory: `ls /home/pi/familywall/server/logs/`
    19. Verify: error.log and output.log files exist

    **Scheduled Restart (INFRA-02):**
    20. Check timer status: `systemctl status kiosk-restart.timer`
    21. Verify: Shows "active (waiting)" status
    22. Check next trigger: `systemctl list-timers --all | grep kiosk`
    23. Verify: Shows next trigger at 03:00:00 tomorrow
    24. Check service file: `systemctl cat kiosk-restart.service`
    25. Verify: ExecStart points to /home/pi/familywall/scripts/restart-kiosk.sh
    26. Manual restart test: `sudo systemctl start kiosk-restart.service`
    27. Observe: Chromium browser should close and reopen within ~10 seconds
    28. Verify: React app reloads successfully after restart
    29. Check restart logs: `journalctl -u kiosk-restart.service -n 50`
    30. Verify: No error messages, shows graceful shutdown sequence

    **Expected outcomes:**
    ✅ Chromium opens full-screen kiosk mode on boot automatically
    ✅ Screen never blanks after 5-10 minutes of inactivity
    ✅ PM2 manages backend with 512MB memory limit visible
    ✅ Backend auto-restarts on crashes
    ✅ systemd timer schedules daily 3am restart
    ✅ Manual restart test completes successfully in ~10 seconds
    ✅ Logs show graceful SIGTERM→SIGKILL sequence

    **Common issues to check if verification fails:**
    - DISPLAY variable not set in systemd service (Chromium fails to start)
    - labwc config in wrong location (using Wayfire path instead)
    - xset commands used instead of labwc idle settings (screen still blanks)
    - PM2 not saved (backend doesn't start on boot)
    - systemd timer not enabled (no scheduled restart)
  </how-to-verify>
  <resume-signal>
    Type "approved" if all verification steps pass, or describe specific failures for troubleshooting.
  </resume-signal>
