  <files>server/ecosystem.config.js</files>
  <action>
    Create server/ecosystem.config.js following PM2 best practices from research. Configure:
    - name: 'familywall-backend'
    - script: './index.js'
    - cwd: '/home/pi/familywall/server' (absolute path for production)
    - instances: 1
    - autorestart: true
    - max_memory_restart: '512M' (user requirement)

    - error_file: './logs/error.log'
    - out_file: './logs/output.log'
    - log_date_format: 'YYYY-MM-DD HH:mm:ss Z'
    - env: { NODE_ENV: 'production', PORT: 3000, DB_PATH: './data/familywall.db' }
    - kill_timeout: 5000 (wait 5s for graceful shutdown before SIGKILL)

    Use exact structure from research Example 2 (PM2 Ecosystem with All Production Settings). This ensures proper memory leak protection and crash recovery per INFRA-04.
