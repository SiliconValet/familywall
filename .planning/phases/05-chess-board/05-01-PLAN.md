   ```sql
   CREATE TABLE IF NOT EXISTS chess_games (
     id INTEGER PRIMARY KEY CHECK (id = 1),
     fen TEXT NOT NULL DEFAULT 'rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1',
     moves TEXT NOT NULL DEFAULT '[]',
     white_player_id INTEGER,

     FOREIGN KEY (white_player_id) REFERENCES family_members(id) ON DELETE SET NULL,

   );
   ```

3. Create server/routes/chess.js with these endpoints following the established chores.js pattern:

   - **GET /api/chess/game** — Returns current game state. If no row exists (id=1), INSERT default starting position then return it. Response: `{ fen, moves: [...SAN strings], white_player_id, black_player_id }`

   - **POST /api/chess/move** — Body: `{ from, to, promotion }`. Load FEN from DB, create Chess instance with that FEN, call chess.move({ from, to, promotion: promotion || 'q' }). If move invalid, return 400 `{ error: 'Invalid move' }`. On success, append move.san to moves JSON array, UPDATE row with new FEN and moves, return `{ fen, move: move.san, moves: [...all moves] }`.

   - **POST /api/chess/undo** — Load game from DB. Parse moves JSON array, if empty return 400. Create Chess instance, replay all moves except the last one to reconstruct FEN. UPDATE row with reconstructed FEN and shortened moves array. Return `{ fen, moves: [...remaining moves] }`.

   - **POST /api/chess/new-game** — UPDATE chess_games SET fen to starting position, moves to '[]', keep player IDs. Return `{ fen, moves: [], white_player_id, black_player_id }`.

   - **PUT /api/chess/player** — Body: `{ color: 'white'|'black', player_id: number|null }`. UPDATE white_player_id or black_player_id. Return updated game state.

   Import chess.js in the route file using: `import { Chess } from 'chess.js';`
   Import db using: `import db from '../db.js';`

4. Register the chess routes in server/index.js by adding:
   ```javascript
   import chessRoutes from './routes/chess.js';
   ```
   and `fastify.register(chessRoutes);` following the existing registration pattern.

2. Create client/src/hooks/useChessGame.ts:
   - Import `{ useState, useEffect, useCallback }` from 'react'
   - Import `{ Chess }` from 'chess.js'
   - Import `{ ChessGame }` from '../types/chess'

   State variables:
   - `const [chess] = useState(() => new Chess())` — single instance
   - `const [fen, setFen] = useState(chess.fen())` — current board position
   - `const [moves, setMoves] = useState<string[]>([])` — SAN move history
   - `const [whitePlayerId, setWhitePlayerId] = useState<number | null>(null)`
   - `const [blackPlayerId, setBlackPlayerId] = useState<number | null>(null)`
   - `const [loading, setLoading] = useState(true)`

   Functions (all useCallback wrapped):
   - `loadGame()` — fetch GET /api/chess/game, chess.load(data.fen), replay moves to set internal state, update all state vars, setLoading(false). Handle case where game doesn't exist yet (API auto-creates).
   - `makeMove(from: string, to: string)` — POST /api/chess/move with { from, to, promotion: 'q' }. On success, chess.move({ from, to, promotion: 'q' }), update fen and moves state. Return true/false for success.
   - `undoMove()` — POST /api/chess/undo. On success, reset chess instance: create new Chess(), replay returned moves array, update fen and moves state.
   - `resetGame()` — POST /api/chess/new-game. On success, chess.reset(), update fen to chess.fen(), clear moves array.
   - `setPlayer(color: 'white' | 'black', playerId: number | null)` — PUT /api/chess/player with { color, player_id: playerId }. Update whitePlayerId or blackPlayerId state.

   useEffect: call loadGame() on mount.

   Return object: `{ fen, chess, moves, whitePlayerId, blackPlayerId, loading, makeMove, undoMove, resetGame, setPlayer }`

   Important: The chess instance is used client-side only for board() method to get piece positions. Server is source of truth for game state via FEN. After each API call, sync local chess instance state with server response.
