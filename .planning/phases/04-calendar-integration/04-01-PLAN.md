2. Add three new tables to server/db.js AFTER the existing CREATE TABLE statements (inside the same db.exec block or a new one):

   ```sql
   CREATE TABLE IF NOT EXISTS calendar_auth (
     id INTEGER PRIMARY KEY CHECK (id = 1),
     google_email TEXT,
     access_token TEXT,
     refresh_token TEXT NOT NULL,
     token_expiry INTEGER,
     created_at INTEGER DEFAULT (unixepoch()),
     updated_at INTEGER DEFAULT (unixepoch())
   );

   CREATE TABLE IF NOT EXISTS calendar_sources (
     id TEXT PRIMARY KEY,
     calendar_id TEXT NOT NULL,
     summary TEXT NOT NULL,
     background_color TEXT,
     selected INTEGER DEFAULT 1,
     family_member_id INTEGER,
     sync_token TEXT,
     created_at INTEGER DEFAULT (unixepoch()),
     updated_at INTEGER DEFAULT (unixepoch()),
     FOREIGN KEY (family_member_id) REFERENCES family_members(id) ON DELETE SET NULL
   );

   CREATE TABLE IF NOT EXISTS calendar_events (
     id TEXT PRIMARY KEY,
     calendar_source_id TEXT NOT NULL,
     summary TEXT,
     description TEXT,
     location TEXT,
     start_time TEXT,
     end_time TEXT,
     start_date TEXT,
     end_date TEXT,
     is_all_day INTEGER DEFAULT 0,
     status TEXT DEFAULT 'confirmed',
     raw_json TEXT,
     created_at INTEGER DEFAULT (unixepoch()),
     updated_at INTEGER DEFAULT (unixepoch()),
     FOREIGN KEY (calendar_source_id) REFERENCES calendar_sources(id) ON DELETE CASCADE
   );

   CREATE INDEX IF NOT EXISTS idx_calendar_events_source ON calendar_events(calendar_source_id);
   CREATE INDEX IF NOT EXISTS idx_calendar_events_start ON calendar_events(start_time);
   CREATE INDEX IF NOT EXISTS idx_calendar_events_start_date ON calendar_events(start_date);
   CREATE INDEX IF NOT EXISTS idx_calendar_sources_selected ON calendar_sources(selected);
   ```

   Key design notes:
   - `calendar_auth` has CHECK (id = 1) to enforce single-account per D-04
   - `calendar_sources.id` is the Google Calendar ID string (e.g., "primary", "user@gmail.com")
   - `calendar_events` stores both `start_time` (ISO8601 dateTime for timed events) and `start_date` (YYYY-MM-DD for all-day events) per D-17
   - `is_all_day` flag distinguishes all-day from timed events per CAL-11
   - `raw_json` stores full Google Calendar event JSON for future use

  <action>
1. Create `server/routes/calendar.js` as a Fastify plugin (same pattern as chores.js) with these endpoints:

   **GET /api/calendar/auth** — Generate OAuth URL (per D-01, D-04)
   - Create OAuth2Client with process.env.GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, OAUTH_REDIRECT_URI
   - Generate auth URL with: `access_type: 'offline'`, `prompt: 'consent'`, `scope: ['https://www.googleapis.com/auth/calendar.readonly']`
   - Return `{ authUrl: string }`

   **GET /api/calendar/oauth/callback** — OAuth callback (exchange code for tokens)
   - Extract `code` from query params
   - Call `oauth2Client.getToken(code)` to exchange for tokens
   - Store refresh_token in calendar_auth table (INSERT OR REPLACE with id=1)
   - Store google_email from token info or userinfo
   - Listen to `oauth2Client.on('tokens', ...)` to persist refreshed access tokens
   - After token storage, fetch calendar list from Google Calendar API
   - For each calendar, INSERT into calendar_sources (id=calendarId, summary=cal.summary, background_color=cal.backgroundColor, selected=1)
   - Auto-map family members: For each calendar_source, query family_members table. If calendar summary (lowercased) contains a family member name (lowercased), set family_member_id to that member's id (per D-05)
   - Redirect to `/#/calendar-connected` (frontend can detect this and show success)

   **GET /api/calendar/status** — Check if calendar is connected
   - Query calendar_auth table for id=1
   - Return `{ connected: boolean, email: string | null }`

   **GET /api/calendar/sources** — List calendar sources (per D-02, D-03, D-06)
   - Query calendar_sources joined with family_members
   - Return array of `{ id, summary, backgroundColor, selected, familyMemberId, familyMemberName }`

   **PUT /api/calendar/sources/:id** — Toggle calendar source selection
   - Update calendar_sources SET selected = request.body.selected WHERE id = params.id
   - Return updated source

   **GET /api/calendar/events** — Fetch cached events for date range
   - Query params: `start` (ISO date), `end` (ISO date)
   - Query calendar_events WHERE calendar_source_id IN (selected sources) AND ((start_time >= start AND start_time < end) OR (start_date >= start AND start_date < end))
   - Join with calendar_sources to get family_member_id and summary
   - Return array of events with family member color info

   **POST /api/calendar/sync** — Trigger sync (manual or auto-refresh per D-08, D-09)
   - Check calendar_auth exists
   - Create OAuth2Client, set credentials with stored refresh_token
   - For each selected calendar_source:
     - If sync_token exists: use incremental sync (events.list with syncToken)
     - If no sync_token: full sync (events.list with timeMin = 30 days ago, timeMax = 90 days ahead, singleEvents: true, maxResults: 2500)
     - Handle HTTP 410: clear sync_token, delete all events for that source, retry full sync
     - For each returned event: UPSERT into calendar_events (INSERT OR REPLACE)
     - For cancelled events (status='cancelled'): DELETE from calendar_events
     - Save new nextSyncToken to calendar_sources.sync_token
   - Return `{ synced: true, eventCount: number, lastSynced: ISO timestamp }`

   **POST /api/calendar/disconnect** — Remove OAuth tokens and cached data
   - DELETE FROM calendar_events
   - DELETE FROM calendar_sources
   - DELETE FROM calendar_auth WHERE id = 1
   - Return `{ disconnected: true }`

   Important implementation details:
   - Create a helper function `getOAuth2Client()` that creates OAuth2Client and loads stored refresh_token from DB
   - Wrap Google API calls in try-catch, return appropriate HTTP errors (401 for auth issues, 502 for Google API failures)
   - Debounce sync: Check calendar_auth.updated_at, skip if last sync was < 30 seconds ago (prevents spam, per D-08 note)
   - Store `updated_at = unixepoch()` in calendar_auth after each successful sync

2. Register the new route in `server/index.js`:
