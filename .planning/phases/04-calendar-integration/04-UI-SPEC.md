## Spacing Scale

Declared values (Tailwind default scale, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 4px | Icon gaps, minimal inline padding, event dot spacing, time indicator line height |
| 2 | 8px | Compact element spacing, internal button padding, event card gaps, event dot size |
| 4 | 16px | Default element spacing, form field gaps, card internal padding |
| 6 | 24px | Section padding, major element separation |
| 8 | 32px | Layout gaps between major sections |
| 12 | 48px | Major section breaks, page-level spacing |
| 16 | 64px | Large page-level spacing, monthly grid cell height |

**Exceptions for this phase:**

- **Touch targets:** Minimum 48px (12 units) for all interactive elements (inherited from Phase 3 standard)
  - View toggle buttons (Daily/Weekly/Monthly): `min-h-12` (48px)
  - Navigation arrows (prev/next): `min-h-12 min-w-12` (48px)
  - Today button: `min-h-12` (48px)
  - Refresh icon button: `min-h-12 min-w-12` (48px)
  - Event cards (tap to expand): `min-h-12` (48px)
  - Monthly grid day cells (tap to navigate): `min-h-16` (64px)

- **Monthly grid cell height:** 64px (16 units, exception to accommodate date number + event dots + touch target)
  - Rationale: Monthly grid cells must fit date number (top), up to 3 event dots (middle), and remain tappable (64px exceeds 48px minimum)

- **Event dot size:** 8px diameter (2 units)
  - Spacing between dots: 4px (1 unit)
  - Rationale: Small enough to fit multiple events per day, large enough to distinguish colors

- **Time indicator line (daily view):** 4px height (1 unit)
  - Color: Primary accent color (oklch(0.52 0.105 223.128))
  - Rationale: Thin line draws attention without dominating layout, 4px maintains grid alignment

**Source:** Inherited from Phase 3 with calendar-specific exceptions approved by user in confirmed decisions.

## Color

Colors use oklch color space (preset configuration). Values shown are CSS custom properties.

| Role | Light Mode Value | Usage |
|------|------------------|-------|
| **Dominant (60%)** | `--background: oklch(1 0 0)` (white) | Page background, main surface, daily/weekly agenda background |
| **Secondary (30%)** | `--card: oklch(1 0 0)` (white), `--muted: oklch(0.97 0 0)` (light gray) | Event card backgrounds, monthly grid cells, all-day event section, settings modal |
| **Accent (10%)** | `--primary: oklch(0.52 0.105 223.128)` (blue) | Reserved for: time indicator line (daily view), Today button (when active), refresh button active state, focus rings |
| **Family Member Colors** | `--chart-1` through `--chart-4` | Calendar event color-coding (left border on event cards, event dots in monthly grid) |
| **Destructive** | `--destructive: oklch(0.577 0.245 27.325)` (red-orange) | Disconnect calendar button only |

**Accent reserved for:**
1. Time indicator line in daily view (4px horizontal line showing current time)
2. Today button (when current period is today)
3. Refresh button active state (when tapped or syncing)
4. Focus ring on form inputs (keyboard navigation)

**NOT used for:** Event cards, navigation arrows, view toggle buttons, monthly grid cells, event dots.

**Family Member Color-Coding (D-21, inherited from Phase 3):**
- Use existing chart color tokens: `--chart-1`, `--chart-2`, `--chart-3`, `--chart-4`
- Automatic mapping by calendar name (D-05): If calendar name contains family member name, assign that member's color; fallback to default color
- Applied to: Left border accent (4px) on event cards, event dots in monthly grid
- **NOT applied to:** Card backgrounds (remain white/muted), time labels (remain foreground)

**Chore Integration (D-31, D-34):**
- Chores displayed in calendar view use same chart color mapping as in Chores tab (left border + assignee color)
- Completed chores fade to 0.7 opacity (inherited behavior from Phase 3)

## Interaction Patterns

### Touch Feedback
All interactive elements provide immediate visual feedback (inherited from Phase 2):

```css
/* Applied to all buttons and tappable elements */
.touch-target:active {
  transform: scale(0.96);
  transition: transform 150ms ease-out;
}
```

**Requirements (FAM-08, inherited):**
- Feedback appears within 100ms of touch
- Visual change is perceivable (scale + opacity or background color shift)
- No hover states (kiosk is touch-only)

### Calendar View Switching Flow

1. User taps "Daily", "Weekly", or "Monthly" toggle button (D-11)
2. Active button changes to accent color background, white text
3. View content transitions immediately (no loading state вҖ” data already loaded)
4. Navigation controls adapt:
   - Daily: Arrows navigate days, period label shows "March 23, 2026"
   - Weekly: Arrows navigate weeks, period label shows "Mar 17вҖ“23, 2026"
   - Monthly: Arrows navigate months, period label shows "March 2026"
5. Empty state adapts to view (if no events)

**Design notes:**
- Toggle buttons: Segmented control style (ToggleGroup), 48px height each
- Active state: Primary accent background, white text
- Inactive state: Muted background, muted foreground text
- Transition: Crossfade between views (200ms opacity transition)

### Navigation Flow

**Previous/Next Period (D-26):**
1. User taps left arrow (previous) or right arrow (next)
2. Period label updates to new period
3. Event list re-renders with new date range
4. Animation: Slide-in from left (previous) or right (next) over 200ms

**Jump to Today (D-28):**
1. User taps "Today" button
2. View resets to current period (current day/week/month based on active view)
3. Period label updates to current date range
4. Today button highlights (accent background) when current period is today
5. Animation: Fade-in over 150ms (no slide, immediate context switch)

**Design notes:**
- Arrow buttons: 48Г—48px touch zones, icon-only (left/right arrows from hugeicons)
- Today button: 48px height, text button
- Period label: 24px heading (text-2xl), centered between arrows, not tappable

### Manual Refresh Flow (D-08, D-10)

1. User taps refresh icon button (top-right corner, 48px touch zone)
2. Refresh icon immediately replaced with spinner (same size)
3. API call to `/api/calendar/sync` (POST)
4. On success:
   - Spinner replaced with refresh icon
   - Event list updates with new data
   - No toast notification (silent success)
5. On failure:
   - Spinner replaced with refresh icon
   - Toast notification: "Unable to sync calendar. Check your connection and try again." with retry button
   - Cached events remain visible

**Design notes:**
- Refresh button: Icon-only (refresh icon from hugeicons), 48Г—48px touch zone
- Spinner: Same size as icon (24px), centered in touch zone
- Debounce: Prevent spam clicks (ignore taps while already syncing)

### Auto-Refresh Flow (D-07, D-09)

1. Background timer triggers every 15 minutes (useInterval hook)
2. Silent API call to `/api/calendar/sync` (POST)
3. On success:
   - Event list updates with new data
   - No UI indication (silent background sync)
4. On failure:
   - No toast notification (silent failure)
   - "Last synced X minutes ago" badge appears (if sync age > 15 minutes)
5. Polling pauses when page not visible (page visibility detection)

**Design notes:**
- No loading state for background refresh (D-09 specifies silent sync)
- Only show "Last synced" badge if sync fails and age exceeds 15 minutes
- Auto-refresh continues until user closes app or device sleeps

### Event Card Interaction (D-19)

**Tap to Expand (D-19):**
1. User taps event card (48px minimum height)
2. Card animates expansion:
   - Height: Auto transition from collapsed (48px) to expanded (auto) over 300ms ease-out (user-approved timing)
   - Content: Description, location, time details fade in during expansion
3. Expanded state shows:
   - Event title (18px, same as collapsed)
   - Time: "[start time] - [end time]" (14px label small)
   - Location: "Location: [location]" (16px body) or hidden if no location
   - Description: "[description]" (16px body) or "No description" if empty
4. Tap again to collapse (same animation reversed)

**Design notes:**
- Only one event card expanded at a time (expanding new card collapses previous)
- Expansion animation: `transition: height 300ms ease-out` (user-approved)
- Collapse animation: Same timing reversed
- Collapsed state: Event title + time only, color bar indicates family member

### Monthly Grid Cell Tap (D-16)

1. User taps day cell in monthly grid (64px height)
2. View switches to Daily view for that date
3. Period label updates to tapped date
4. Event list shows that day's events
5. Animation: Fade transition over 150ms

**Design notes:**
- Entire cell is tappable (64px height, full width)
- Active state: Scale(0.98) on tap (subtle feedback)
- Cell contains: Date number (14px, top-left), event dots (8px, centered below date)

### Calendar Settings Flow (D-01, D-04, D-06)

**Open Settings (PIN-gated):**
1. User taps settings icon (48px touch zone)
2. PIN modal appears (reuses Phase 2 PinModal component)
3. User enters 4-digit PIN
4. On success: Calendar Settings modal opens
5. Modal shows:
   - Connected account email (or "No calendar connected")
   - "Connect Calendar" button (if not connected) OR "Disconnect" button (if connected)
   - Calendar selection list (checkboxes for each calendar, only if connected)
   - Family member color mapping table (read-only, only if connected)

**Connect Calendar (OAuth Flow, D-04):**
1. User taps "Connect Calendar" button
2. API call to `/api/calendar/auth` (GET) returns OAuth URL
3. OAuth URL opens in new window (or same window on kiosk)
4. User completes Google OAuth consent screen
5. Callback redirects to `/api/calendar/oauth/callback`
6. On success:
   - Settings modal refreshes showing connected account
   - Calendar selection list appears
   - All calendars selected by default
   - Auto-sync begins (initial sync with "Syncing calendar..." spinner)
7. On failure:
   - Toast notification: "Calendar connection failed. Please try again or check your Google account permissions."
   - Settings modal remains open

**Select Calendars (D-02, D-03):**
1. User toggles checkboxes for each calendar
2. Changes save immediately (API call to `/api/calendar/sources`)
3. Event list re-renders with updated calendar selection
4. Color mapping updates (automatic name matching, D-05)

**Disconnect Calendar:**
1. User taps "Disconnect" button
2. Confirmation dialog appears: "Disconnect Calendar?" / "Events will no longer sync. You can reconnect anytime."
3. User taps "Disconnect" or "Keep Connected"
4. On Disconnect:
   - API call to `/api/calendar/disconnect` (POST)
   - Settings modal closes
   - Calendar view shows empty state: "No Calendar Connected"
   - All cached events removed

**Design notes:**
- Settings modal: Full-screen overlay, vertically scrollable
- Calendar selection: Each calendar shows name (18px body large) + checkbox (48px)
- Family member mapping: Table format (read-only, informational only)
  - Columns: Calendar Name | Family Member | Color (visual swatch)
  - Auto-populated based on name matching (D-05)

### Chores Integration in Calendar View (D-31, D-33, D-34, D-35)

**Unified Timeline (D-31):**
1. Calendar view queries both `/api/calendar/events` AND `/api/chores` endpoints
2. Merge events + chores by time (events use `start.dateTime`, chores use `due_date` or `created_at`)
3. Render unified list sorted by time (ascending)
4. Chores display with checkbox (same as Chores tab, 48px)
5. Event cards display without checkbox (read-only)

**Chore Completion in Calendar View (D-34):**
1. User taps chore checkbox (48px touch zone)
2. Same completion flow as Chores tab:
   - Checkbox animates (scale 0 вҶ’ 1.2 вҶ’ 1.0, 300ms ease-out)
   - Chore card fades out over 200ms (D-35)
   - Toast notification: "Chore completed" with 5-second "Undo" button
3. Completed chores remain in calendar view (faded to 0.7 opacity, D-35)
4. Chore stays in merged timeline (does not move to separate section)

**Design notes:**
- Chore cards in calendar view: Same styling as Chores tab (left border accent, points badge)
- Completed chores: Opacity 0.7, checkbox filled with assignee color
- Chores tab remains separate (D-32): Focused task completion, no events shown
- Calendar view is "context view": Shows events + chores together for unified family schedule

## State Management

### Component States
Each interactive element supports these states (inherited from Phase 2):

| State | Visual Treatment | When Active |
|-------|------------------|-------------|
| Default | Standard colors from design tokens | Element is idle |
| Hover | Not used (touch-only device) | N/A |
| Focus | Ring: `--ring` color, 2px offset | Keyboard navigation (secondary) |
| Active/Pressed | `scale(0.96)`, slight opacity shift | Touch down (before release) |
| Disabled | Opacity 50%, cursor not-allowed | Form validation fails, loading state |
| Loading | Spinner replaces icon/text, disabled state | Async operation in progress (sync) |
| Error | Border: destructive color, error text below | Sync failure, OAuth failure |
| Success | Transient toast notification | Sync success (manual only) |

### Calendar-Specific States

| State | Visual Treatment | When Active |
|-------|------------------|-------------|
| Calendar Not Connected | Empty state: "No Calendar Connected" | No OAuth token stored |
| Initial Sync | Full-page spinner: "Syncing calendar..." | First sync after OAuth connection |
| Manual Refresh | Spinner replaces refresh icon | User tapped refresh button |
| Background Refresh | Silent (no UI) | Auto-refresh every 15 minutes |
| Sync Failed | "Last synced X minutes ago" badge | Sync age > 15 minutes and no active refresh |
| Event Card Collapsed | Title + time only, 48px min height | Default state for event cards |
| Event Card Expanded | Full details (location, description), auto height | User tapped event card |
| Today Active | Today button highlighted (accent background) | Current period is today |
| View Active | View toggle button highlighted (accent background) | Current view mode (Daily/Weekly/Monthly) |

### View States
- **Daily View:** Show events for current day (6AM-10PM timeline)
- **Weekly View:** Show events for current week (7 columns, Mon-Sun)
- **Monthly View:** Show events for current month (calendar grid, all days visible)
- **Empty state adapts to view:**
  - Daily: "No Events Today" / "Your schedule is clear. New events will sync automatically."
  - Weekly: "No Events This Week" / "You're free for [date range]. Events will appear when scheduled."
  - Monthly: No empty state (grid always shows all days)
- **Not connected state (all views):** "No Calendar Connected" / "Connect your Google Calendar in Settings to see events here."

## Motion and Animation

### Transitions (Inherited from Phase 2)
- Button press: `transform 150ms ease-out`
- Modal enter: `opacity 200ms ease-out`, `scale 200ms ease-out`
- Modal exit: `opacity 150ms ease-in`, `scale 150ms ease-in`
- Toast enter: `slide-in-right 300ms ease-out`
- Toast exit: `fade-out 200ms ease-in`

### Calendar-Specific Animations

**Event Card Expansion (User-Approved):**
- Tap to expand: `height auto` transition with 300ms ease-out
- Collapse: Same timing reversed
- Content fade: Description/location fade in during expansion (opacity 0 вҶ’ 1 over 200ms)

**View Switching:**
- View content: Crossfade between views (opacity 0 вҶ’ 1 over 200ms)
- No slide animation (immediate context switch)

**Navigation (Previous/Next):**
- Event list: Slide-in from left (previous) or right (next) over 200ms
- Period label: Update immediately (no animation)

**Jump to Today:**
- Event list: Fade-in over 150ms (no slide)
- Today button: Highlight immediately when period is today

**Loading States (User-Approved):**
- Initial sync: Full-page spinner with "Syncing calendar..." text (fade-in 150ms)
- Manual refresh: Spinner replaces refresh icon (crossfade 150ms)
- Background refresh: Silent (no animation)

**Monthly Grid Cell Tap:**
- Active state: Scale(0.98) over 150ms
- Transition to Daily view: Fade transition over 150ms

**No Animations:**
- Background auto-refresh (silent)
- "Last synced" badge appearance (instant)
- Calendar settings changes (instant update)

**Performance:** All animations use `transform` and `opacity` (GPU-accelerated). No `left`, `top`, `width`, or `height` animations except for event card expansion (acceptable for user-initiated action).

## Responsive Considerations

**Target device:** Raspberry Pi touchscreen (800Г—480 or 1024Г—600 typical resolutions)

**Design approach:** Fixed layout optimized for kiosk screen size. Not responsive to different viewports.

**Minimum assumptions:**
- Width: 800px minimum
- Height: 480px minimum
- Touch input only (no mouse hover)
- Landscape orientation (primary)

**Portrait mode:** Not prioritized for Phase 4. If device rotates, layout should remain functional but may not be optimized.

**Scrolling:**
- Daily view: Vertically scrollable if event list exceeds viewport
- Weekly view: Horizontally scrollable if 7 columns exceed viewport width (unlikely on 800px+ screens)
- Monthly grid: Vertically scrollable if grid exceeds viewport (unlikely with 64px cell height)
- Calendar settings modal: Vertically scrollable if content exceeds viewport height

**View-Specific Layout:**
- Daily view: Single column, vertical scroll
- Weekly view: 7-column grid, horizontal scroll if needed
- Monthly view: 7Г—5-6 grid, fits within viewport (no scroll expected)

## Accessibility

### Touch Target Compliance
- All interactive elements: Minimum 48Г—48px (inherited from Phase 3 standard, exceeds WCAG 2.2 Level AAA 44px requirement)
- Event cards: Minimum 48px height (tap to expand)
- Monthly grid cells: 64px height (exceeds minimum, user-approved exception)
- Navigation buttons: 48Г—48px (arrows, refresh, settings)
- View toggle buttons: 48px height each
- Adequate spacing between targets: Minimum 8px gap

### Keyboard Navigation
Not primary input method (kiosk is touch-only), but supported:
- Tab order follows visual order (top to bottom, left to right)
- Focus indicators: Ring visible on all focusable elements
- Enter key activates focused buttons
- Escape key closes modals
- Arrow keys navigate between event cards (optional enhancement)

### Color Contrast
- Body text on background: 21:1 (AAA)
- Event time labels on card: Muted foreground on white (minimum 4.5:1)
- Primary button text on primary: Verified contrast in preset
- Destructive button text on destructive: Verified contrast in preset
- Family member chart colors: Verified sufficient contrast for left border accent (not primary text color)
- Event dots in monthly grid: High contrast against white cell background

### Semantic HTML
- Buttons use `<button>` element (not div with onClick)
- Calendar grid uses `<table>` or semantic grid (role="grid")
- Event cards use proper ARIA attributes (role="article", aria-label with event title)
- View toggles use ToggleGroup with proper ARIA roles
- Time indicator uses `<hr>` or semantic divider with ARIA label "Current time"
- Settings modal uses proper ARIA attributes (dialog role, aria-labelledby, aria-describedby)

### Screen Reader Support
- Event cards announce: "[event title], [time range], [location if present]"
- Monthly grid cells announce: "[date number], [N events]"
- Navigation buttons announce: "Previous day/week/month", "Next day/week/month", "Today"
- View toggles announce: "Daily view", "Weekly view", "Monthly view"
- Sync status announces: "Last synced X minutes ago" or "Syncing calendar"
